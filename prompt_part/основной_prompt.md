Описание целей и задач, приближённо, ввдение:
Нужно составить интенсивный план развития для Сергея Герасина на 3 месяца.
Сейчас он "middle-" - типовые задачи тянет, но на архитектуре, отладке и самостоятельном проектировании буксует. Soft skills ниже ожидаемого. Цель - за 3 месяца довести до уверенного Middle+. Формат жёсткий: каждая неделя должна быть расписана.

Что должен покрывать план:

1. TypeScript (продвинутый уровень). Дженерики, utility types (Partial, Pick, Omit, Record, Extract, Exclude), conditional types и infer, template literal types, mapped types, рекурсивные типы. Type guards, discriminated unions, exhaustive checks. Строгая типизация API-контрактов (request/response/errors). Декораторы. Работа с tsconfig в strict mode. Цель - не просто знать синтаксис, а писать код, где TS реально ловит баги до рантайма.

2. Nuxt 4 (глубокое погружение). Архитектура: layers, модули, плагины - когда что применять. Composables: паттерны, переиспользование, тестирование. Server routes: Nitro, H3, серверные middleware, кэширование, API-валидация (zod/valibot). Рендеринг: SSR, SSG, ISR, SPA - выбор стратегии, гидратация, payload-оптимизация. Состояние: Pinia + useState, SSR-безопасность, серверное vs клиентское состояние. Route и server middleware. SEO: useHead, useSeoMeta, OG-теги, JSON-LD. Обработка ошибок: error boundaries, createError, showError, error.vue. Авторизация и защита роутов. WebSocket через Nitro. Работа с nuxt.config на продвинутом уровне: runtime config, app config, hooks.

3. Сборка, тулинг и инфраструктура. Vite: конфигурация, плагины, dev-server, HMR, кастомные плагины. Rollup: chunks, tree-shaking, side effects, manual chunks. Анализ бандла: vite-bundle-visualizer, оптимизация размера. Nitro: presets (node, vercel, cloudflare, edge workers), storage, KV, кэш-стратегии. PostCSS, Tailwind - конфигурация и оптимизация. ESLint flat config, Prettier, lint-staged, husky. Docker: контейнеризация Nuxt-приложения. CI/CD: пайплайн сборки, тесты, деплой. Мониторинг: Sentry, логирование ошибок на фронте.

4. Тестирование. Vitest: unit-тесты для composables, утилит, store. Vue Test Utils / @nuxt/test-utils: компонентные тесты. Playwright или Cypress: E2E-тесты ключевых сценариев. MSW (Mock Service Worker) для мока API. Стратегия тестирования: что тестировать, что нет, пирамида тестов.

5. Производительность и оптимизация. Lighthouse: метрики, аудит, исправление. Core Web Vitals: LCP, FID/INP, CLS - понимание и оптимизация. Lazy-loading компонентов и роутов. Оптимизация изображений: nuxt/image, srcset, форматы. Prefetching, preloading. Профилирование рендеринга через Vue DevTools. Мемоизация, computed vs watch, предотвращение лишних ре-рендеров.

6. AI, агенты и LLM в разработке. Эффективное использование AI-ассистентов: GitHub Copilot, Cursor, Claude Code - промпт-инженерия, контекст, ограничения. Основы LLM: токены, контекстное окно, температура, модели. AI-агенты: концепция, MCP (Model Context Protocol), автоматизация рутины. Практика: интеграция AI-чата в Nuxt-приложение через API (Anthropic/OpenAI), стриминг ответов, обработка ошибок. RAG - базовое понимание паттерна. Vercel AI SDK / аналоги. Ревью AI-сгенерированного кода: галлюцинации, безопасность, качество.

7. Soft skills. Коммуникация: статусы задач, блокеры, участие во встречах, внятные описания PR. Code review: давать конструктивный фидбек, а не просто approve. Декомпозиция задач и оценка сроков. Документирование решений и ADR (Architecture Decision Records). Инициативность: предлагать улучшения, провести минимум 1 доклад/демо для команды. Умение задавать правильные вопросы вместо «я застрял».

Формат плана:

Разбить на 3 месяца, каждый месяц - по неделям. На каждую неделю: фокусная тема, конкретные активности (что изучить, что сделать на практике), критерий выполнения. Обучение должно быть привязано к реальным рабочим задачам - не абстрактные курсы, а «взял фичу X и сделал её с применением Y». Можно взять на примере одного из наших крупных проектов и ставить по ним внутр. задачи в рамках обучения, пробовать применять знания и контролировать ревью/работу.

Приложить: ресурсы для обучения по каждой теме (курсы - их можно брать с coursehunters, статьи, документации), график 1-on-1 (раз в неделю при таком темпе), чекпоинт в конце каждого месяца с ревью прогресса и корректировкой плана.

Внутри рабочего заказа сделать ряд эпиков по обучению, где будет практика, cамо обучение для Сергея, с разбитыми темами, описанием и ожидаемым результатом. Запланировать на себя и меня активности по ревью и контролю. Все эпики связать связью к общему обучению.

Предварительный план согласовать со мной в рамках задачи и закрепить его в конфе в карточке сотрудника.

Примерное распределение фокуса:

Месяц 1 - фундамент: продвинутый TypeScript, глубокий Nuxt 4 (архитектура, SSR, composables), тестирование, soft skills с первого дня.

Месяц 2 - инфраструктура и качество: сборка/тулинг, CI/CD, производительность, продолжение Nuxt (Nitro, middleware, state), тестирование на практике.

Месяц 3 - AI и закрепление: AI-инструменты и интеграция, Docker, мониторинг, финальный проект, объединяющий все темы. Простой доклад для команды по результатам.

С первого дня нужно заложить обучение примитивным AI-фичам. Понимание разных систем и моделей, адекватный промптинг, подключение copilot/claude и базовая работа с ним. Уметь обучаться вместе с ними, распрашивать и писать код вместе с ними.

В дополнении к этому стоит учесть, что:
Тестирование:
Его можно перенести в самый конец или даже за рамки этих 3 месяцев, так как сейчас мы его фактически не применяем. Предлагаю сосредоточиться на мануальном тестировании и профилировании результатов своей работы, а также на оценке производительности и оптимизации.
Цель: на выходе получать качественно протестированные задачи, уметь анализировать результат работы (какие запросы выполняются, сколько весят, сколько времени занимают и т.д.), работать с профайлингом и DevTools:
измерять скорость при разных условиях интернета
оценивать производительность
выявлять утечки памяти и т.п.
В общем, фокус на мануальном тестировании + DevTools.

TypeScript:
Предлагаю разделить обучение на две части:
Общая (в начале): обычные типы, утилитарные типы, интерфейсы, проверки и гарды - то, что входит в ежедневную рутину.
Расширенная (ближе к концу): инферы, дженерики, вычисляемые типы и другие продвинутые возможности. На этом этапе можно дать попробовать реализовать небольшую библиотеку и полностью покрыть её типами по стандартам.

После TypeScript:
Сразу делаем акцент на понимание инструментов Vue/Nuxt.

Культура обучения:
Важно формировать T-shape культуру: чтобы человек понимал рынок, популярность технологий, зачем нужны одни инструменты и почему другие устаревают, какие есть разновидности инструментов, их преимущества и недостатки.

GraphQL и WebSockets:
Нужно дать понимание работы с ними с нуля:
какие задачи они решают
когда их стоит использовать, а когда нет
альтернативные пути и их плюсы/минусы

Архитектура и паттерны:
К SOLID и архитектурным подходам добавляем: KISS, DRY, YAGNI, MVC, MVVM, Flux, Clean, FSD, дизайнерские паттерны.
Важно не просто знать их, а понимать:
что это такое
когда и зачем применять
какие решения лучше подходят при разной сложности проекта
И всё это подкреплять реальными примерами и практикой. Иногда «наговнокодить» может быть оптимальным выбором, и это тоже нужно понимать.

Прозрачность и измеримость софтов:
На протяжении всего обучения внедряем понятные и измеримые результаты и фиксируем их в плане обучения.

Фокус проектов:
Хочется уйти от примеров с дашбордами и сосредоточиться на более разнообразных проектах: скин, проэксперт, LDB и другие.

Детализация плана:
На этапе детализации необходимо разбить всё по неделям с фиксированным планом обучения, практикой и промежуточными результатами. По итогам каждого блока - более детальная встреча в формате собеса
