ЭТАП 2 — Практика TypeScript + Vue + Nuxt

Цель: Научиться применять TypeScript, Vue и Nuxt в реальных задачах
проекта.

Темы: - Архитектура Nuxt (layers, plugins, modules, composables) -
Server API (Nitro, server routes, middleware) - State management (Pinia,
useState) - SSR особенности (hydration, server/client state)

Практика: Задача 1 — реализовать API endpoint (server/api/orders.ts) с
типизацией. Задача 2 — реализовать composable useOrders(). Задача 3 —
реализовать страницу pages/orders.vue.

Проверка: Тимлид проверяет архитектуру, типизацию, читаемость кода и
структуру Nuxt.

Тестовое задание: Сделать виджет dashboard с загрузкой данных,
отображением и обработкой ошибок.

Критерии оценки: 5 — правильная архитектура, корректная типизация,
чистый код. 4 — рабочий код, мелкие ошибки. 3 — архитектурные проблемы.
<3 — возврат к обучению.

ЭТАП 3 — Сборка, инфраструктура и тестирование

Цель: Понять как работает сборка, тестирование и запуск приложения.

Темы: - Vite (dev server, HMR, plugins) - Rollup (chunks, tree
shaking) - Docker (контейнеризация) - CI/CD (pipeline сборки) -
Тестирование (Vitest, Playwright)

Практика: 1. Проанализировать bundle проекта через
vite-bundle-visualizer. 2. Написать unit тесты для composables и utils.
3. Написать e2e тест через Playwright. 4. Собрать docker образ проекта.

Тестовое задание: Подготовить приложение к деплою: - docker - тесты -
pipeline описание

Критерии оценки: 5 — глубокое понимание сборки и тестирования. 4 —
работает, но есть пробелы. 3 — поверхностное понимание. <3 — повтор
этапа.

------------------------------------------------------------------------

ЭТАП 4 — Теория тестирования

Цель: Понять стратегии тестирования и когда какие тесты использовать.

Темы: - тестовая пирамида (unit / integration / e2e) - стратегии
тестирования - mocking (MSW) - testability

Практика: Определить слабые места. Предложить стратегию тестирования.

Тестовое задание: Написать тесты для новой фичи: - unit - integration -
e2e

Критерии оценки: 5 — грамотная тестовая стратегия. 4 — рабочие тесты. 3
— слабые тесты. <3 — повтор этапа.

------------------------------------------------------------------------

ЭТАП 5 — AI инструменты

Цель: Освоить использование AI как инструмента разработки.

Темы: - основы LLM (токены, контекст) - Copilot / Cursor - RAG - AI
агенты

Практика: 
1. Генерация типов и тестов через AI. 
2. Создание AI чата в Nuxt. 
3. Анализ ошибок AI.

Тестовое задание: Реализовать AI чат: - отправка запроса - стриминг
ответа - обработка ошибок

Критерии оценки: 5 — грамотное использование AI. 4 — работает, но
поверхностно. 3 — не понимает ограничений AI. <3 — повтор этапа.
