### Тип задачи

Базовое погружение в TypeScript (runtime → compile-time безопасность)

### Материалы

https://www.typescriptlang.org/docs/handbook/intro.html  
https://www.typescriptlang.org/docs/handbook/utility-types.html  
https://www.typescriptlang.org/tsconfig

### Продолжительность

### Практическое задание

Создать чистый проект на typescript.
Использовать строгую типизацию в рамках проекта проекта.
Тебе требуется описать сущности, которые приходят с бэкенда используя базовые подходы typescript.
А также регламентировать дальнейшую работу с данными и запрос на сервер.

С сервера может приходить несколько статусов об ошибке: invalid_data, invalid_key, unauthenicated, unknown
Струкутура ошибок с бэкенда при ошибке { data: 'invalid_key' - или любой другой ключ из выше указанных }
Структура успешного ответа { data: {
calendar: {
from: '23-09-2022',
before: '24-09-22'
}
isShow: true,
dates: [
{ key: 1, month: 1, day: 1, year: 2022 },
{ key: 2, month 1, day 2, year: 2022 }
]

} }

Нельзя использовать `any`.

### Теоретическое задание (для интервью) - для лида

Что такое:

- structural typing
- type inference
- union vs intersection
- optional vs undefined

### Теоретическое задание (подготовить)

Благодаря какому подходу TypeScript способен находить баги до runtime. - Опиши подход и расскажи как представляешь себе работу, особенности typescript в проекте до и после сборки.
